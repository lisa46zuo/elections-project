Introduction

Clustering is a type of machine learning that is used to identify groups of similar objects, or clusters, in a dataset. There are different methods used for clustering, including partition clustering and hierarchical clustering.

Partition clustering is a type of clustering that tries to group objects in a dataset into a specified number of clusters (the k-value). The most popular partition clustering method is k-means clustering, where the cluster is represented by the means of the data points. It is more sensitive to outliers than PAM clustering, another type of partition clustering.

The procedure for k-means clustering is as follows:

  1. Select the number of clusters (k) you want to identify in the data. This can be done using an elbow plot. For this example we will use 3 clusters.
  2. Randomly select 3 distinct data points as intial clusters.
  3. Measure the distance from a point to the initial clusters.
  4. Assign the point to the nearest cluster.
  5. Repeat for the rest of the data points.
  6. Calculate the mean of each cluster.
  7. Repeat measure and cluster using the mean values.
  8. Assess the quality of clustering by adding up variation within each cluster.
  9. Repeat the whole process using different points as initial clusters.
  10. Pick the best clustering.

Calculating the distance between rows creates a distance matrix. There are different ways to measure distance:

  • Euclidean distance is taking the square root of the sum of each difference from one row to another for each column: \(\sqrt{(\mbox{difference}_1)^2 + (\mbox{difference}_2)^2 + (\mbox{difference}_3)^2...}\)

  • Manhattan distance is adding the absolute value of the difference between the rows: \(|\mbox{difference}_1| + |\mbox{difference}_2| + |\mbox{difference}_3| + ...\)

  • Correlation-based distances are ways to measure distance using different correlation coefficients. These include Pearson correlation distance, Eisen cosine correlation distance, Spearman correlation distance, and Kendall correlation distance. Correlation-based distances use correlation to determine distance, so objects that are far away when calculated with Euclidean or Manhattan distance may be considered close to each other with correlation-based distances if they are highly correlated.

Hierarchical clustering is a clustering method that identifies groups without having a k-value specified. Hierarchical clustering orders rows and columns of a dataset based on similarity so it is easier to see correlations within the dataset. Hierarchical clustering is associated with heatmaps, which are charts that show the relative intensity with which each variable is expressed. For example, heatmaps are used to find the relationship between species based on the expression of genes:

The procedure for hierarchical clustering is as follows, using a heatmap:

  1. Figure out which row is most similar to the first row (distance).
  2. Figure out which row is most similar to the second row.
  3. Continue for the rest of the rows.
  4. Find the most similar pair of rows, and merge the two rows into a cluster.
  5. Repeat but treat treat the new cluster as a single row.
  6. Merge rows into clusters until there is only one cluster left.

There are different ways to compare points and clusters. Clusters can be compared using the centroid, single-linkage, or complete-linkage methods. Using the centroid, points are compared to the mean of each cluster. Using single-linkage, points are compared to the closest point in each cluster. Finally, using complete-linkage, the point is compared to the farthest point in each cluster.

Methods

This week we are going to use a small dataset of life expectancy in the 1960s in several different cities around the world. The data was used found on John Hartigans website and was used in his book Clustering Algorithms. In the original dataset, columns included the year the data was recorded in each city and the number of years of life left for males and females at age 0, 25, 50, and 75:

knitr::kable(head(data, 5))

Montreal 1966 67 44 22 7 73 51 28 9
Toronto 1966 68 45 23 8 75 53 29 10
Vancouver 1966 68 46 23 8 75 53 30 11
Kaohsiung 1966 66 43 22 8 71 49 26 9
Taipei 1966 69 46 24 8 72 50 27 9

To prepare the data for cluster analysis, the year column was removed, since all of the data was recorded in the 1960s and so differences in years should be insignificant. In addition, the life expectancy were changed to total life expectancy at each age instead of number of years of life left, so data at each age is easier to compare.

data <- data[, 2:9] # drop year column
data$M25 <- data$M25+25 # adjust so life expectancy is total life expectancy, not years left
data$M50 <- data$M50+50
data$M75 <- data$M75+75
data$F25 <- data$F25+25
data$F50 <- data$F50+50
data$F75 <- data$F75+75
